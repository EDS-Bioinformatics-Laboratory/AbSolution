# AbSolution Roadmap

This document outlines the planned directions for AbSolution development. It is intended to provide transparency about future priorities and to invite community input. Timelines are approximate and may shift based on funding, feedback, and contributions.

Suggestions and discussion are welcome via [GitHub Issues](https://github.com/EDS-Bioinformatics-Laboratory/AbSolution/issues).

---

## Ongoing

- **CRAN maintenance and versioning.** Active maintenance of the CRAN release with semantic versioning, bug fixes, and compatibility updates for new R versions and dependencies.
- **Documentation and vignettes.** Continuous improvement of user documentation, including expanded vignettes, usage examples, and tutorials covering common analysis workflows.
- **Zenodo archival.** Persistent, citable archival of each release on Zenodo, linked to the ENCORE-structured file system snapshot (sFSS).

## Short-Term

- **Extended input format support.** Broaden compatibility with additional AIRR-seq resources following standard data formats.
- **Enhanced visualization options.** Additional plot types and improved interactivity for exploratory analysis, including expanded accessibility features.
- **Training materials.** Development of step-by-step tutorials and workshop materials to support adoption in educational and research settings.

## Medium-Term

- **New analysis modules.** Integration of additional analytical methods for receptor clonal analysis.
- **Single-cell data integration.** Support for single-cell immune profiling datasets, enabling joint analysis of transcriptomic and repertoire data.

## Long-Term

- **ENCORE generalization.** Further development and promotion of the ENCORE framework as a standalone resource for reproducibility in interactive applications beyond AbSolution, with documentation and examples for adoption in other domains.
- **Community-driven extensions.** Support for modular, community-contributed analysis plugins through a standardized extension interface.

---

## How to Get Involved

If you are interested in contributing to any of these directions, or if you have suggestions for the roadmap, please see [CONTRIBUTING.md](CONTRIBUTING.md) or open an issue.
